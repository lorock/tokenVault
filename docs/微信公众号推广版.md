# 你的 2FA 密钥，可能正被偷偷同步到别人服务器

扫码收个验证码，看起来是随手一点的事。

但如果我告诉你——你扫进验证器的那串密钥，可能已经被悄悄上传到了第三方服务器，你还觉得安全吗？

## 这个坑，很多人没意识到

二次验证（2FA）的设计初衷是：**你知道密码，你拥有设备**。密钥只存在你的设备上，别人就算拿到密码，没设备也进不去。

但现实里，不少验证器为了"方便多端同步"，把密钥直接传到了自家云端。

**等于你把家门钥匙复印了一份，寄存在别人家里。**

他们服务器被拖库，你的密钥就跟着泄露；他们拿密钥做什么，你完全不知道。条款里写"加密存储"——加密的是传输还是存储？密钥谁控制？这两个问题，没几个产品敢正面答。

## 所以我做了一个不一样的：令牌盒

核心就一句话：**密钥从不出设备。**

| 维度 | 主流验证器（含大厂） | 令牌盒 |
|------|---------------------|--------|
| 密钥存储 | 本地 + **云端同步** | **仅本地** |
| 加密方式 | 云端说了算 | **AES-256-GCM**，你自设主密码 |
| 断网可用 | 依赖同步状态 | **完全离线**，飞机模式照常出码 |
| 开源审计 | ❌ 闭源黑盒 | ✅ **MIT 开源**，代码全公开 |
| 使用方式 | 装 App | **浏览器打开即用**，无需安装 |

密钥用 **AES-256-GCM 信封加密**，主密码经 **PBKDF2 25 万次迭代**派生密钥；解锁后数据密钥只驻留内存，锁屏即清除。

你可以理解为：密钥被锁在一个本地保险箱里，密码只有你知道。没有云端、没有后台、没有服务器知道你存了什么。

## 纯本地，不代表难用

- **浏览器直达**：打开就能用，不用装 App
- **扫码录入**：上传二维码图片（微信相册 / 电脑文件都行），自动识别 `otpauth://`
- **实时刷新**：倒计时进度条，临近刷新由黄变红提醒
- **备份导出**：一键导出 JSON，换设备导入即可
- **分享二维码**：把站点转成可扫的码，发给同事扫码添加
- **生物识别**：指纹 / 面容直接解锁，不用每次输密码

## 开源可审计，不是说说

代码已公开（MIT 协议），关键路径都能自己验证：`totp.js`（标准算法）、`vault.js`（加密保险库）、`useVault.js`（解锁态管理）、`test/`（自动化测试）。算法走 RFC 标准，加密用浏览器原生 Web Crypto，构建产物是纯静态文件，可部署到任意 HTTPS 托管。
